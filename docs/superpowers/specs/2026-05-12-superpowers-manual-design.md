# 设计文档：Superpowers 使用手册与最佳实践

## 背景

基于 `topic/ai/好用的扩展/开发工作流/superpower/` 下的 3 篇已有笔记和 GitHub 仓库 `obra/superpowers` 的 14 个 skill，整理一份面向技术社区的完整使用手册。

## 目标读者

技术社区的 AI 辅助开发者——既需要入门教学，也需要进阶最佳实践。

## 交付物

单篇 MOC 式长文，存放于：
`topic/ai/好用的扩展/开发工作流/superpower/Superpowers 使用手册与最佳实践.md`

## 内容范围

### 覆盖范围

- 核心工作流 6 个 skill（深度讲解）
- 进阶 skill 4 个（紧凑介绍）
- 自定义 skill 入门
- 最佳实践速查
- Skill 速查表

### 不覆盖

- 其他 Agent 平台（Codex、Gemini 等）的具体安装步骤
- Skill 源码分析
- [[Superpowers]] 插件系统的内部实现

## 组织方式

线性旅程式——按一个完整开发流程的顺序组织，贯穿一个虚拟场景。

### 虚拟场景

给一个已有的 Next.js Todo 应用添加"导出为 Markdown"功能。

选择原因：
- 足够简单，读者能快速理解需求
- 涉及前后端、有设计决策空间
- 完整触达核心工作流的每个阶段

## 教学模式

### 核心章节（第 1-6 站）

统一四层结构：

| 层次 | 内容 | 目的 |
|------|------|------|
| 为什么 | skill 解决什么问题，没有它会怎样 | 建立动机 |
| 怎么运作 | skill 的内部流程（[[Mermaid]] 或步骤列表） | 理解机制 |
| 场景演示 | Agent 与人类对话，用 callout 呈现 | 看到实际样子 |
| 最佳实践 | 人类 partner 应该做什么、常见陷阱 | 实战指导 |

### 进阶章节

紧凑格式：
- 一句话核心价值
- 触发时机
- 3-5 个关键要点
- 与已有笔记的交叉引用

## 写作风格

- 简体中文，术语保留英文原文
- 对话式但不轻浮——像有经验的同事讲解
- 避免翻译腔

## 格式约定

| 元素 | 处理方式 |
|------|----------|
| Agent 对话 | `> [!agent]` / `> [!user]` callout |
| 关键概念 | 首次出现加粗并附简短说明 |
| 命令/代码 | 代码块，标注语言标识符 |
| 流程图 | [[Mermaid]] 图 |
| 警告/提示 | `> [!warning]` / `> [!tip]` callout |
| 最佳实践 | `> [!best-practice]` callout |
| 反模式 | `> [!caution]` callout |

## 与已有笔记的关系

- 不重复已有内容，用 `> 相关笔记：[[xxx]]` 引导
- 补充已有笔记没有的内容（skill 内部机制、人类 partner 指南、陷阱）
- 整合 Jesse 博客中的设计理念作为引言素材
