## 服务有多少个？

测试服务器当前运行 **10 个容器**（infra + drawing-cad），另外还有 upda-cloud、drawing-pdf、drawing-ofd 等模块尚未部署。

完整的 Docker Compose 定义了约 **20+ 个服务**，涵盖前端、后端、中间件、数据库四个层次。

## 端口分配一眼看懂

```
170xx → 前端服务（用户直接访问）
180xx → 后端服务（Gateway 路由/API）
181xx → 中间件（Nacos/MinIO/RabbitMQ/Qdrant 等）
190xx → 数据库（MySQL/MongoDB/Redis）
```

这个约定让团队成员看到端口号就知道是哪类服务，减少沟通成本。
