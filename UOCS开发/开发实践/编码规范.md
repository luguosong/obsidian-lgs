# 编码规范

> **权威来源**：本文件。`CLAUDE.md` 和 `copilot-instructions.md` 均引用此处，不重复维护。
> 修改编码规范时只改本文件。

## 后端规范（Spring Boot / Spring Cloud）

- 构造器注入：`@RequiredArgsConstructor` + `final` 字段，禁止 `@Autowired`
- DTO 命名：`{动作}{实体}ReqDTO` / `RespDTO`
- API 文档：使用 `@Tag`、`@Operation`、`@Schema`
- ORM：[[MyBatis]]-Plus + XML Mapper
- [[异常处理]]：禁止 `catch` 后静默忽略，必须记录或显式抛出
- 包路径：`com.upda.{api|services|common}.{module}.{controller|service|entity|mapper|config}`

## 前端规范（uocs-client）

- 仅使用函数式组件 + Hooks
- 使用 MobX 管理状态
- 网络请求统一经 `services/` 与 `axios-factory`（已内联至 `src/util/`）
- 图标使用内联的 `src/iconfont/`（`@ant-design/icons` 除外）
- UI 组件库为 MUI v7（`@mui/material` + `@mui/lab` + `@mui/x-*`）

## 前端规范（中间件查看器应用）

| 项目 | 目录 | 技术栈 | 端口 |

- 中间件查看器应用彼此独立，与 `upda-cloud` 无代码依赖

## Entity 变更联动规则

[[Entity]] 实体类发生任何变更时，**必须按顺序执行 5 步**：

1. 生成增量 DDL，并在用户确认后执行到开发环境数据库
2. 更新 `src/main/resources/schema.sql`
3. 更新 `src/main/resources/mapper/{Entity}Mapper.xml`
4. 新增实体时补齐 `{Entity}Mapper.java`
5. 检查相关 `ReqDTO` / `RespDTO`

服务→数据库映射：

- `services-basic` → `uocs-basic`
- `service-ca` → `uocs-ca`
- `service-seal` → `uocs-seal`
- `service-signer` → `uocs-signer`
- `service-ra` 无独立数据库

> 详细 DDL 操作流程、数据库连接与实体对照表见 `upda-cloud/upda-cloud-server/CLAUDE.md`。

## 文档命名规范

- `docs/` 下 Markdown 默认中文命名，技术缩写可保留
- `references/` 不受此约束

## 全局禁止事项

- 禁止在 `upda-code/` 根目录执行 `git init` / `git add`
- 禁止修改本地 `application.yml` 承载业务配置
- 禁止硬编码数据库密码、连接生产数据库
- 禁止手动修改 `docker-compose/data/`
- 禁止静默执行 DDL，必须先确认
- 禁止无关重构
- 禁止在 `uocs-client` 中添加已迁移服务（services-gd、services-open-cloud）相关页面
- 禁止直接修改 `drawing-cad/drawingweb/` 中的代码。需要修改时，在 `docs/` 下生成修改报告（含文件路径、修改内容、原因），由用户通知相关人员执行
