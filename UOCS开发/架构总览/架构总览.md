## 项目概述

**UOCS 云平台**（绿建科技）— 企业级综合性云平台服务。整体分为两大部分：

- **UPDA Cloud**（`upda-cloud/`）：后端 Spring Cloud 微服务 + 前端 React + Vite，当前核心业务覆盖 CA 证书管理（三级 PKI + OCSP + TSA）、RA 证书签发代理、电子印章全生命周期（含注册编排）、CMS 多重签名、AI RAG 对话，后续将持续扩展更多云服务
- **中间件查看器平台**（`drawing-*/`）：独立于微服务体系的文件查看/编辑中间件，含 Web CAD（`drawing-cad/`）、PDF（`drawing-pdf/`）、OFD（`drawing-ofd/`）、UOCS 文档（`drawing-uocs/`）四个子项目

## 请求流程

```
浏览器 → Gateway → AuthGlobalFilter(JWT验证)
  → 注入 X-User-Id / X-User-Roles / X-User-Scopes / X-Token-Type 头
  → Nacos 路由到目标微服务
  → UserInterceptor → UserContext(ThreadLocal)
  → ScopeInterceptor(@RequireScope) → PermissionInterceptor(@RequirePermission)
  → Controller → Service → Mapper(DB) / Feign Client
```

## UPDA Cloud 服务拓扑

- `auth`：[[OAuth2]] 授权服务器（Authorization Server / OIDC / 联合登录 / 第三方应用管理 / [[Token Exchange]] RFC [[8693]]）
- `service-basic`：用户 / RBAC / MinIO
- `service-ca`：证书引擎（签发 / 吊销 / OCSP / TSA / CRL）
- `service-ra`：纯 PKI RA 服务，仅负责身份核验与证书操作，无数据库
- `service-seal`：印章全生命周期与注册编排
- `service-signer`：独立签章服务（[[第三方集成]] / 文件代理 / 回调 / [[会话管理]]），依赖 service-seal 和 service-ca

### 模拟业务系统（根目录 demo-biz-system/，独立运行，不参与生产部署）

- `demo-biz-system`：模拟第三方业务系统，验证 Signer **Mode A（托管）和 Mode B（自助）** 对接流程（H2 内存数据库，端口 9090），已从 `upda-cloud-server/test-services/` 迁出，作为独立 [[Spring Boot]] 项目运行（`mvn spring-boot:run`）。包含 HMAC-SM3（BouncyCastle）签名工具、[[Token Exchange]] 客户端、回调验签控制器。

> 对接文档：`docs/Oauth改造/OAuth2认证体系设计与实现.md`（含原理、Checklist、Mode A/B 参考实现代码、常见陷阱）

## 中间件查看器平台（drawing-*/）

### drawing-cad/（Web CAD 图纸平台）

- `drawing-web-app`：React 19 + MobX 6 + Ant Design 6 + Vite 7 + ODA WASM（DrawingJs），Web CAD 查看/编辑器 + AI 智能体前端（:3000）。内置 **[[License Gate]]** 前端（LicenseStore + LicenseErrorOverlay + WASM 握手）与 GIS 服务面板（MapServer/FeatureServer 接入）
- `drawing-ai-server`：[[Spring Boot]] 3.4 + [[Spring AI]] 1.1 + WebFlux，CAD AI 后端，采用 **[[Harness Engineering]] 架构**（QueryEngine 执行内核 + ToolProtocol 元数据 + PromptAssembler 分层装配 + ContextManager 自动压缩 + SessionMemory + Multi-Agent 审图编排 + 错误恢复韧性），兼容 Vercel AI SDK UI Message Stream 协议（:3001）；同时承载 **[[License Gate]] 后端**（Bouncy Castle SM2 + OSHI 硬件指纹 + `/api/license/*` 4 endpoint + `LicenseToolsMain` 厂商离线 CLI），已完全取代早期 Node.js License 服务
- RAG 知识库：[[Spring AI]] + Qdrant 向量数据库，内置于 `drawing-ai-server`（集合 `drawing_knowledge`，宿主机端口 18140/18141）
- WASM 源码：`drawing-cad/drawingweb/`（C++ Emscripten 绑定源码），覆盖 CadCore 主入口、70+ OdDb 实体类、几何/图形/颜色/运行时类、全量枚举与 [[CadCodeExecutor 沙箱]]上下文

> 详细架构见 `drawing-cad/drawing-web-app/CLAUDE.md` 和 `drawing-cad/drawing-ai-server/CLAUDE.md`。

### drawing-pdf/（PDF 查看器）

- `drawing-pdf-app`：React 19 + MUI 7 + MobX 6 + pdfjs-dist，PDF 文件查看器（:5200），含 SM2/SM3 签名验证模块

### drawing-ofd/（OFD 查看器）

- `drawing-ofd-app`：React 19 + MUI 7 + MobX 6 + jszip，OFD 文件查看器前端（:5201），含 SM2/SM3 签名验证模块，**已接入 UOCS [[OAuth2]] PKCE 认证**（authStore + pkce.js），通过 UOCS Token 调用 drawing-ofd-server 印章 API
- `drawing-ofd-server`：[[Spring Boot]] 3 + Spring Security [[OAuth2]] Resource Server，OFD 签章后端（:5212），验证 [[Bearer Token]]（JWKS: `:18011/oauth2/jwks`），代理调用 UOCS Gateway（:18010）印章接口，CORS 允许来源 `:5201`

### drawing-uocs/（已迁移模块暂存）

- 从 `upda-cloud/upda-cloud-server` 迁出的代码模块：`services-open-cloud`（OpenCloud .NET 容器代理）、`services-gd`（广东省定制）、`api-open-cloud`、`api-gd`
- 这些模块的 POM 父引用仍指向原后端项目，需后续独立化
- 对应的 Docker 服务定义已从 `compose.yaml` 移除（server-open-cloud、server-gd、gd\_business-server、gd-business-client、gd-batch-signing-client）
- .NET OpenCloud 容器（open-cloud）仍保留在 Docker 中，作为文档协作中间件运行

## 关键边界

- `service-ra` 不再负责注册审批流程，注册编排已迁移至 `service-seal`
- `service-seal` 的证书操作调用链：`RaClient → service-ra → service-ca`
- `service-ca` 的 `/internal/**` 仅供内部 Feign 调用，Gateway 拦截外部访问
- `gmcore-server / gmcore-client` 为遗留独立系统，禁止与 UOCS 新功能耦合
- `services-open-cloud` 和 `services-gd` 代码已迁移至 `drawing-uocs/`，对应 Docker 服务已清理，后端不再与 OpenCloud 交互
