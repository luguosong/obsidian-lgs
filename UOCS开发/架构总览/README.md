# 架构总览

> 这是理解 UOCS 平台的起点。先读项目简介建立全局概念，再查架构总览了解细节。

## 为什么需要先看架构？

UOCS 不是单体应用——它是一个由 **7 个后端微服务 + 3 个独立前端 + 多个中间件** 组成的分布式系统。在深入任何子项目之前，你需要先理解：

1. **请求怎么流转的** — 从浏览器到 Gateway 到各个微服务
2. **服务怎么通信的** — Feign 调用、Nacos 注册发现、MQ 消息
3. **代码怎么组织的** — 哪些是 Git 仓库，哪些只是分类目录
