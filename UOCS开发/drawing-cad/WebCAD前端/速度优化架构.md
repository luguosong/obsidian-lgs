# 速度优化架构

DrawingWebApp 前端性能关键实现细节。

---

## DrawingStateCache（前端缓存）

`services/DrawingStateCache.js` — 脏标记模式缓存，避免每次发消息都遍历全图实体。

- **读取**：`getDrawingStateCached(viewer, Module)` 返回缓存或重新计算
- **失效**：7 个写操作（modify/create/delete/transform/undo/executeCode/executeSCR）调用 `invalidateDrawingState()`
- **零值保护**：当 WASM 迭代器在异步上下文中返回 0 实体时，返回 `null` 而非零值结果

---

## DwgResultSummarizer（前端摘要）

`services/DwgResultSummarizer.js` — 替代原始 JSON 截断，为每种读取类型提供结构化摘要。

- `summarize(readType, jsonData)` — 按类型分发到专用摘要器
- 支持 6 种类型：structure / text / blocks / entities / dimensions / layers
- `_safeName()` 辅助函数处理对象/字符串/undefined 等边界情况

---

## MarkdownRenderer（前端渲染）

`components/AiAssistant/MarkdownRenderer.jsx` — react-markdown + remark-gfm 渲染 AI 响应。

- 自定义 `p` 组件防止 `<pre>` 嵌套在 `<p>` 中
- 暗/亮主题样式（CSS Modules）
- 表格、代码块、链接自定义渲染

---

## feedbackTrigger 竞态保护

`AiAssistant.jsx` 中的 `feedbackTrigger` state 防止 marker 跳过递归后的竞态条件：

- marker 跳过递归 → SSE 快速结束 → `status='ready'` 先于 WASM 执行完成 → feedback useEffect 空跑
- WASM 执行完成后 `setFeedbackTrigger(prev => prev + 1)` 重触发 useEffect

---

## RightSidebar 覆盖面板

`components/RightSidebar/RightSidebar.jsx` — CSS overlay 方式实现面板，**避免 WebGL 黑屏**。

- **根因**：改变 canvas 容器宽度 → ResizeObserver → `canvas.width=` → 清空 WebGL 缓冲区
- **方案**：`.panelArea` 使用 `position: absolute` 覆盖在 canvas 上方，canvas 宽度不变
- 拖拽调整宽度通过 CSS 变量 `--rightPanelWidth` 实现，localStorage 持久化
