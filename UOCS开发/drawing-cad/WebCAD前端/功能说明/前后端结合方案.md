完全兼容，而且用 [[Spring AI]] 做后端反而更合适。

## 架构调整

原方案是纯前端直连 LLM API，换成 [[Spring AI]] 后端后：

```
前端 (React)
    ↓ HTTP/SSE
Spring AI 后端
    ↓ Function Calling
LLM (OpenAI/Claude/通义)
    ↓ 返回命令序列
Spring AI 后端解析
    ↓ SSE 流式推送
前端 CommandExecutor
    ↓
Viewer.execute()
```

### 定义 CAD 工具函数

```java
@Component
public class CadDrawingTools {

    @Tool(description = "绘制直线")
    public CadCommand drawLine(
        @ToolParam(description = "起点X坐标") double x1,
        @ToolParam(description = "起点Y坐标") double y1,
        @ToolParam(description = "终点X坐标") double x2,
        @ToolParam(description = "终点Y坐标") double y2
    ) {
        return new CadCommand("LINE", List.of(
            x1 + "," + y1,
            x2 + "," + y2,
            ""
        ));
    }

    @Tool(description = "绘制圆形")
    public CadCommand drawCircle(
        @ToolParam(description = "圆心X") double cx,
        @ToolParam(description = "圆心Y") double cy,
        @ToolParam(description = "半径") double radius
    ) {
        return new CadCommand("CIRCLE", List.of(cx + "," + cy, String.valueOf(radius)));
    }

    @Tool(description = "绘制矩形")
    public CadCommand drawRectangle(
        @ToolParam(description = "左下角X") double x1,
        @ToolParam(description = "左下角Y") double y1,
        @ToolParam(description = "右上角X") double x2,
        @ToolParam(description = "右上角Y") double y2
    ) {
        return new CadCommand("RECTANG", List.of(
            x1 + "," + y1,
            x2 + "," + y2
        ));
    }

    @Tool(description = "绘制多段线")
    public CadCommand drawPolyline(
        @ToolParam(description = "坐标点列表，格式: x1,y1;x2,y2;...") String points,
        @ToolParam(description = "是否闭合") boolean closed
    ) {
        List<String> args = new ArrayList<>(Arrays.asList(points.split(";")));
        args.add(closed ? "C" : "");
        return new CadCommand("PLINE", args);
    }
}
```

### SSE 流式接口

```java
@RestController
@RequestMapping("/api/ai/cad")
public class CadAiController {

    private final ChatClient chatClient;
    private final CadDrawingTools cadTools;

    public CadAiController(ChatClient.Builder builder, CadDrawingTools cadTools) {
        this.cadTools = cadTools;
        this.chatClient = builder
            .defaultSystem("""
                你是一个 AutoCAD 绘图助手。
                用户用自然语言描述图形，你调用工具绘制。
                坐标未指定时合理推算，复杂图形分解为基本命令。
                """)
            .build();
    }

    @GetMapping(value = "/draw", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> draw(@RequestParam String prompt) {
        return chatClient.prompt()
            .user(prompt)
            .tools(cadTools)
            .stream()
            .content()
            .map(chunk -> ServerSentEvent.builder(chunk).build());
    }
}
```

## 前端对接

```javascript
// services/CadAiService.js
export async function sendCadPrompt(prompt, onCommand, onText) {
    const es = new EventSource(`/api/ai/cad/draw?prompt=${encodeURIComponent(prompt)}`);
    
    es.onmessage = (e) => {
        const data = JSON.parse(e.data);
        
        if (data.type === 'command') {
            // LLM 调用了工具，执行绘图命令
            onCommand(data.command);  // { name: 'LINE', args: ['0,0', '100,0', ''] }
        } else if (data.type === 'text') {
            // 文字回复
            onText(data.content);
        }
    };
    
    es.onerror = () => es.close();
}
```

```javascript
// CommandExecutor.js
export async function executeCommand(viewer, command) {
    await viewer.execute(command.name, command.args);
}
```

## 关键点

**Tool 返回值的问题**：[[Spring AI]] 的 `@Tool` 方法返回值会作为 tool result 回传给 LLM，但你实际上需要把命令推送给前端。有两种处理方式：

**方式一**：Tool 只组装命令，Controller 拦截 tool call 结果推送 SSE（推荐，控制精确）

**方式二**：Tool 直接返回 JSON 字符串，前端解析整个对话流中的结构化片段

方式一需要自定义 `ToolCallResultConverter` 或监听 `ChatResponse` 中的 `ToolCall` 事件，[[Spring AI]] 目前支持这个扩展点。

## 总结

[[Spring AI]] 后端比纯前端方案更好：
- API Key 安全
- 可以加会话记忆（`ChatMemory`）
- 可以加 RAG（命令手册知识库）
- Function Calling 的 Tool 定义比前端更干净

核心验证点不变：**先确认 `viewer.execute()` 能否批量传参驱动命令**。
