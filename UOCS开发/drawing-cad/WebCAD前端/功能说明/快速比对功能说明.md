# 快速比对功能 — 执行流程与原理说明

## 整体架构概览

快速比对是一个**像素级图纸差异比对**功能，采用三层协作架构：

| 层级 | 文件 | 职责 |
|------|------|------|
| **UI 层** | `ReviewRibbon.jsx` | 按钮交互、文件选择 |
| **JS 服务层** | `ViewerService.js`（Viewer 类） | 文件上传到虚拟 FS、WebGL 比对着色器、渲染循环调度 |
| **C++ 引擎层** | `CadCore.cpp` | 打开第二张图纸、创建第二个渲染设备、同步视图参数 |

---

## 详细执行流程

### 第 1 步：用户点击"快速比对"按钮

**文件：** `ReviewRibbon.jsx` 第 119–141 行

"工具"标签页中渲染了一个隐藏的 `<input type="file" accept=".dwg,.dxf">`，以及一个"快速比对"按钮。
点击按钮触发 `handleStartCompare`，它通过 `ref` 模拟点击隐藏的文件输入框，弹出系统文件选择对话框（只接受 `.dwg` / `.dxf` 文件）。

```jsx
const handleStartCompare = useCallback(() => {
  if (compareInputRef.current) compareInputRef.current.click()
}, [])
```

### 第 2 步：用户选择文件 → `handleCompareFileChange`

**文件：** `ReviewRibbon.jsx` 第 47–54 行

获取到 `File` 对象后，调用 Viewer 实例的 `startCompare(file)` 方法。
若返回成功，将 MobX store 中的 `isCompareMode` 设为 `true`，UI 切换为显示"退出比对"按钮。

```jsx
const handleCompareFileChange = useCallback(async (ev) => {
  const file = ev.target.files?.[0]
  if (!file || !viewerRef.current) return
  ev.target.value = ''
  const ok = await viewerRef.current.startCompare(file)
  if (ok) store.setCompareMode(true)
  else console.warn('快速比对启动失败')
}, [viewerRef, store])
```

### 第 3 步：JS 层 `startCompare` — 文件上传 + 调用 C++

**文件：** `ViewerService.js` 第 1232–1263 行

这里做了三件关键事情：

1. **文件读取**：用 `FileReader` 将用户选择的 DWG 文件读取为 `ArrayBuffer`
2. **写入 Emscripten 虚拟文件系统**：通过 `Module.FS_createDataFile` 将文件写入路径 `/__compare_<filename>`
3. **调用 C++ 引擎**：`this.appCore.StartCompare(vpath)` 通过 Embind 调用到 C++ 的 `CadCore::StartCompare`
4. **初始化比对 WebGL 着色器**：`this._initCompareGL()`

```js
async startCompare(file) {
  // 1. FileReader → ArrayBuffer → Uint8Array
  const arraybuffer = await new Promise((resolve) => {
    const reader = new FileReader()
    reader.onload = (ev) => resolve(ev.target.result)
    reader.readAsArrayBuffer(file)
  })
  const array = new Uint8Array(arraybuffer)

  // 2. 写入 Emscripten 虚拟文件系统
  const vpath = '/__compare_' + name
  Module.FS_createDataFile('/', '__compare_' + name, array, true, true, true)

  // 3. 调用 C++ 引擎加载比对图纸
  const ok = this.appCore.StartCompare(vpath)
  if (!ok) return false

  // 4. 初始化 WebGL 比对着色器
  this._compareGL = this._initCompareGL()
  if (!this._compareGL) { this.appCore.ExitCompare(); return false }

  this._compareMode = true
  return true
}
```

### 第 4 步：C++ 层 `CadCore::StartCompare` — 打开第二张图并创建独立渲染设备

**文件：** `CadCore.cpp` 第 1271–1323 行

关键步骤：

1. **读取比对文件**：用 `exHostAppServices.readFile()` 将 DWG 文件解析为 `OdDbDatabase` 对象（`m_pCompareDb`）
2. **创建第二个 GsDevice**：通过 `m_pGs->createDevice()` 创建独立渲染设备（`m_pCompareDevice`），注意 `CreateContext` 设为 `false`（不创建新的 GL 上下文，共用同一个 Canvas）
3. **配置渲染上下文**：背景色和调色板与主设备保持一致
4. **建立布局视图**：`OdDbGsManager::setupActiveLayoutViews` 为比对数据库设置活动布局视图
5. **同步视口尺寸和视图参数**：将主设备的尺寸同步到比对设备，并调用 `SyncCompareView()` 同步相机参数

```cpp
bool CadCore::StartCompare(std::wstring filename)
{
    // 1. 读取比对 DWG 文件
    OdString odName(filename.c_str());
    m_pCompareDb = exHostAppServices.readFile(odName);

    // 2. 创建第二个渲染设备（共享 GL 上下文）
    m_pCompareDevice = m_pGs->createDevice();
    OdRxDictionaryPtr pProperties = m_pCompareDevice->properties();
    pProperties->putAt("CreateContext", OdRxVariantValue(false));

    // 3. 配置渲染上下文
    m_CompareContext.setDatabase(m_pCompareDb);
    m_CompareContext.enableGsModel(true);
    m_pCompareDevice->setBackgroundColor(bgColor);
    m_pCompareDevice->setLogicalPalette(pColor, 256);

    // 4. 建立布局视图
    m_pCompareDevice = OdDbGsManager::setupActiveLayoutViews(m_pCompareDevice, &m_CompareContext);

    // 5. 同步尺寸和视图参数
    m_pCompareDevice->onSize(rc);
    SyncCompareView();

    m_bCompareMode = true;
    return true;
}
```

### 第 5 步：`SyncCompareView` — 同步视图参数

**文件：** `CadCore.cpp` 第 1333–1349 行

将主视图的 6 个相机参数完全复制到比对视图，确保两张图在完全相同的视角下渲染：

- 相机位置（position）
- 目标点（target）
- 上方向向量（upVector）
- 视野宽度（fieldWidth）
- 视野高度（fieldHeight）
- 投影模式（perspective / parallel）

```cpp
void CadCore::SyncCompareView()
{
    OdGsView* pSrc = m_pDevice->viewAt(0);
    OdGsView* pDst = m_pCompareDevice->viewAt(0);

    pDst->setView(
        pSrc->position(), pSrc->target(), pSrc->upVector(),
        pSrc->fieldWidth(), pSrc->fieldHeight(),
        pSrc->isPerspective() ? OdGsView::kPerspective : OdGsView::kParallel
    );
}
```

### 第 6 步：JS 层 `_initCompareGL` — 初始化比对 Shader

**文件：** `ViewerService.js` 第 1081–1162 行

创建一个专用的 WebGL Shader Program，用于在 GPU 上进行像素级差异比对。

**核心 Fragment Shader 逻辑：**

```glsl
precision mediump float;
uniform sampler2D uTexA;      // 主图纸渲染结果
uniform sampler2D uTexB;      // 比对图纸渲染结果
uniform float uThreshold;     // 差异阈值（0.02）
varying vec2 vTexCoord;

void main() {
    vec4 a = texture2D(uTexA, vTexCoord);
    vec4 b = texture2D(uTexB, vTexCoord);
    float diff = length(a.rgb - b.rgb);   // 计算 RGB 颜色向量的欧氏距离

    if (diff < uThreshold) {
        // 像素相同 → 输出灰度（去除颜色干扰，突出差异区域）
        float gray = dot(a.rgb, vec3(0.299, 0.587, 0.114));
        gl_FragColor = vec4(gray, gray, gray, 1.0);
    } else {
        // 像素不同 → 输出纯红色（醒目标注差异）
        gl_FragColor = vec4(1.0, 0.0, 0.0, 1.0);
    }
}
```

同时创建以下 WebGL 资源：

- 全屏四边形顶点缓冲（quad buffer）
- 两个纹理对象 `texA`、`texB`（分别存放两张图的渲染像素）
- 两个像素数组 `pixelsA`、`pixelsB`（CPU 侧缓冲区，用于 `readPixels` 截取）

### 第 7 步：渲染循环中的比对逻辑 — `update()`

**文件：** `ViewerService.js` 第 145–158 行

每帧渲染循环中（由 `requestAnimationFrame` 驱动），如果处于比对模式，执行以下流程：

```js
update() {
    if (this._compareMode && this._compareGL) {
        const res = this._compareGL

        // 1. 渲染主图纸 → 截取像素 A
        this.appCore.Update()
        this._capturePixels(res, 'pixelsA')

        // 2. 渲染比对图纸 → 截取像素 B
        this.appCore.UpdateCompare()
        this._capturePixels(res, 'pixelsB')

        // 3. GPU 比对合成
        this._uploadAndCompare(res)
        return
    }
    // ...正常渲染逻辑...
}
```

**每帧执行 5 个步骤：**

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1 | `appCore.Update()` | C++ 渲染主图纸到 Canvas（WebGL 帧缓冲） |
| 2 | `_capturePixels('pixelsA')` | `gl.readPixels` 截取当前帧缓冲区像素 |
| 3 | `appCore.UpdateCompare()` | C++ 同步视图参数后渲染比对图纸到同一 Canvas |
| 4 | `_capturePixels('pixelsB')` | `gl.readPixels` 截取比对图纸的像素 |
| 5 | `_uploadAndCompare()` | 将两份像素上传为纹理，用比对 Shader 全屏绘制差异结果 |

C++ 端的 `UpdateCompare` 每帧都会先调用 `SyncCompareView()` 确保用户平移/缩放后两个视图保持同步：

```cpp
void CadCore::UpdateCompare()
{
    if (!m_pCompareDevice.isNull() && m_bCompareMode) {
        SyncCompareView();
        m_pCompareDevice->update();
    }
}
```

### 第 8 步：退出比对

当用户点击"退出比对"按钮时，调用链如下：

**UI 层**（`ReviewRibbon.jsx`）：

```jsx
const handleExitCompare = useCallback(() => {
    if (viewerRef.current) viewerRef.current.exitCompare()
    store.setCompareMode(false)
}, [viewerRef, store])
```

**JS 服务层**（`ViewerService.js`）：

```js
exitCompare() {
    this._compareMode = false
    this.appCore.ExitCompare()     // 通知 C++ 释放资源
    this._cleanupCompareGL()        // 清理 WebGL Shader/纹理/缓冲
    this.appCore.Update()           // 恢复正常单图渲染
}
```

**C++ 引擎层**（`CadCore.cpp`）：

```cpp
void CadCore::ExitCompare()
{
    m_bCompareMode = false;
    m_pCompareDevice = NULL;    // 释放比对渲染设备
    m_pCompareDb = NULL;        // 释放比对数据库
    if (!m_pDevice.isNull())
        m_pDevice->invalidate(); // 使主设备重绘
}
```

**JS 层 `_cleanupCompareGL`** 释放 WebGL 资源：

```js
_cleanupCompareGL() {
    const { gl, program, quadBuf, texA, texB } = this._compareGL
    gl.deleteProgram(program)
    gl.deleteBuffer(quadBuf)
    gl.deleteTexture(texA)
    gl.deleteTexture(texB)
    this._compareGL = null
}
```

## 核心原理总结

### 数据流图

```
用户选择DWG文件
        │
        ▼
  FileReader → ArrayBuffer → Emscripten FS 虚拟文件
        │
        ▼
  C++ CadCore::StartCompare()
    ├── readFile() 解析第二个 OdDbDatabase
    ├── createDevice() 创建第二个 GsDevice（共享 GL 上下文）
    ├── setupActiveLayoutViews() 建立布局视图
    └── SyncCompareView() 同步相机参数
        │
        ▼
  JS _initCompareGL() 创建比对 Shader Program
        │
        ▼
  每帧渲染循环 update():
    ┌─ appCore.Update()        → 渲染主图 → readPixels → pixelsA
    ├─ appCore.UpdateCompare() → 同步视图 → 渲染比对图 → readPixels → pixelsB
    └─ GPU Shader 比对:
         对每个像素: diff = length(A.rgb - B.rgb)
         diff < 0.02 → 灰色（无变化）
         diff ≥ 0.02 → 红色（有差异）
```

### 比对算法

- **输入**：两张图纸在相同视角下的渲染像素（RGBA 格式）
- **方法**：对每个像素，计算两图 RGB 颜色向量的欧氏距离
- **阈值**：0.02（归一化 [0,1] 空间，约 5/255 的色差）
- **输出**：
  - 相同区域 → 灰度化（使用 ITU-R BT.601 亮度公式：`0.299R + 0.587G + 0.114B`）
  - 差异区域 → 纯红色 `(1.0, 0.0, 0.0)`

### 设计亮点

1. **共享 GL 上下文**：比对设备设 `CreateContext=false`，两个 ODA 渲染设备向同一个 WebGL 上下文输出，避免多上下文切换开销
2. **GPU 加速比对**：差异计算完全在 Fragment Shader 中并行执行，每像素 O(1) 复杂度，充分利用 GPU 并行能力
3. **实时同步视图**：每帧 `SyncCompareView` 将主视图的 6 个相机参数复制到比对视图，用户可以自由平移/缩放查看不同区域的差异
4. **视觉反馈直观**：相同区域灰度化处理（去除颜色干扰），差异区域用醒目的红色标注
5. **窗口尺寸同步**：`CadCore::Resize` 中同时更新 `m_pCompareDevice` 的尺寸，保证窗口大小改变后比对仍然正确
