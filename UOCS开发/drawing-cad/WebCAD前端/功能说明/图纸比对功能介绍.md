# WebUACAD 图纸比对功能

WebUACAD 提供两种互补的图纸比对能力，覆盖从全局视觉校验到精确对象审查的完整工作流。

---

## 功能概览

| | 像素级比对 | 对象级比对 |
|---|---|---|
| **比对原理** | 渲染两张图纸，逐像素计算颜色差异 | 基于 Handle 句柄，逐对象比较数据库差异 |
| **检测粒度** | 任意视觉差异（含标注文字、线宽、颜色变化） | 对象级变更（新增、删除、修改） |
| **输出形式** | 差异区域着色叠加图 | 结构化变更列表 + 着色叠加图 |
| **适用场景** | 快速发现"哪里改了" | 精确追溯"改了什么" |
| **底层技术** | WebGL Fragment Shader + GPU 并行计算 | ODA TfRevisionControl + XRef 可视化 |

---

## 像素级比对

### 简介

像素级比对采用 GPU 加速的图像差异算法，将两张 DWG 图纸在相同视角下渲染后，逐像素计算颜色差异。无论是几何变更、标注修改、线宽调整还是颜色变化，只要视觉上存在差异就能检测到。

### 核心特点

- **零遗漏检测**：任何视觉可见的改动都会被捕获，不依赖对象结构，不存在盲区
- **GPU 实时计算**：差异检测在 WebGL Fragment Shader 中并行执行，每帧可处理数百万像素
- **实时交互**：支持平移、缩放查看差异，视图同步丝滑流畅
- **三色差异显示**：相同区域灰度化，原图独有内容与比对图独有内容分别着不同颜色，一目了然
- **参数可调**：
  - **差异阈值**：控制检测灵敏度（0.005–0.2），适应不同精度需求
  - **圆弧精度**：调整曲线细分段数（100–5000），消除曲线渲染精度差异导致的误报
  - **自定义配色**：相同区域、原图差异、修改图差异的颜色均可自定义
- **切换比较**：一键切换原图与比对图的叠加顺序，从不同角度审视差异
- **刷新重算**：放大到细节后可重新生成（Regen），获得更精确的比对结果

### 工作原理

```
DWG-A（当前图纸）──→ ODA 渲染引擎 ──→ 像素数组 A ─┐
                                                     ├→ GPU Shader 逐像素比对 → 差异可视化
DWG-B（比对图纸）──→ ODA 渲染引擎 ──→ 像素数组 B ─┘
```

每帧渲染时，两个 ODA GsDevice 在同一 WebGL 上下文中交替渲染，通过 `gl.readPixels` 截取像素后上传为纹理，由 Fragment Shader 对每个像素计算 RGB 颜色向量的欧氏距离：

- 距离 < 阈值 → 像素相同 → 灰度化显示
- 距离 ≥ 阈值 → 像素不同 → 根据来源着色（原图色 / 修改图色）

### 适用场景

- **设计变更快速校验**：快速确认两版图纸之间是否存在差异以及差异分布
- **出图前复核**：确保最终出图与审批版本完全一致
- **建筑/规划图比对**：支持大范围图纸的全局差异扫描
- **非结构化修改检测**：文字内容修改、标注样式调整、图层颜色变化等不影响几何的改动

## 对象级比对

### 简介

对象级比对基于 ODA Transaction Framework（TfRevisionControl），在数据库层面逐对象比较两张 DWG 图纸的差异。通过 Handle 句柄匹配，精确识别每个对象的新增、删除和修改状态，并以结构化列表和着色叠加的方式呈现结果。

### 核心特点

- **精确到对象**：基于 Handle 句柄一一对应比较，准确区分新增、删除、修改三种变更类型
- **结构化结果**：输出包含 Handle、变更类型、实体类型、图层的详细变更列表
- **点击定位**：在结果列表中点击任意条目，视图自动缩放定位到该对象所在位置
- **分类筛选**：支持按变更类型（全部/新增/删除/修改）筛选结果，快速聚焦关注内容
- **着色可视化**：
  - 绿色（ACI 3）— 新增的对象
  - 红色（ACI 1）— 删除的对象（通过 XRef 叠加显示）
  - 蓝色（ACI 5）— 修改的对象
  - 浅灰色（ACI 9）— 未变更的对象（降低视觉干扰）
- **参数可配**：四种状态的 ACI 颜色索引均可通过界面调整
- **版本管理基础**：底层使用 ODA TfRevisionControl，为后续集成图纸版本管理奠定了技术基础

### 工作原理

```
DWG-A（当前图纸）──→ OdTfRepository::store() ──→ 数据库快照 SHA-A ─┐
                                                                      ├→ getRevisionDifference()
DWG-B（比对图纸）──→ OdTfRepository::store() ──→ 数据库快照 SHA-B ─┘
                                                                      │
                                                                      ▼
                                                              ChangeList
                                                    { handle → added/deleted/modified }
                                                                      │
                                              ┌───────────────────────┼───────────────────────┐
                                              ▼                       ▼                       ▼
                                       新增实体着绿色          修改实体着蓝色          删除实体通过XRef
                                       （当前图中）           （当前图+XRef）          红色叠加显示
```

1. **数据库快照**：将两个 DWG 数据库分别存入内存中的 TfRepository，生成 SHA 摘要
2. **差异计算**：`getRevisionDifference()` 比较两个快照，输出以 Handle 为键的变更列表
3. **XRef 叠加**：将比对文件作为外部参照（XRef）加载到当前图中，使删除的对象在原位置以红色显示
4. **着色标注**：遍历变更列表，对当前图中的新增/修改实体修改颜色，未变更实体置灰
5. **图层锁定**：锁定所有图层防止误操作，退出比对时自动恢复

### 结果面板功能

| 功能 | 说明 |
|---|---|
| **变更摘要** | 工具条直接显示 `+新增 -删除 ~修改` 统计数 |
| **详情面板** | 弹出式面板，展示完整变更列表 |
| **分类筛选** | 全部 / 新增 / 删除 / 修改 四个筛选标签 |
| **列表字段** | Handle（句柄）、状态（着色标签）、实体类型、图层 |
| **点击定位** | 点击列表行，视图自动缩放至该对象周围 |

### 适用场景

- **设计审查**：精确追溯每个对象的变更历史，辅助设计审批
- **版本对比**：两版图纸之间的结构化差异分析，生成变更清单
- **协同设计**：多人协作时精确了解他人的修改内容
- **竣工图核验**：竣工图与设计图的逐对象核对

---

## 两种比对的协作使用

推荐的工作流程：

1. **先用像素级比对**快速扫描 — 全局查看差异分布，确认修改范围
2. **再用对象级比对**精确审查 — 逐对象查看变更详情，点击定位逐一核实

两种比对模式互斥使用，退出一种后方可进入另一种，确保操作清晰、状态不混乱。
