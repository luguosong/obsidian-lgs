## 前端对接原理详解

### 整体数据流

```
用户输入 → EventSource → SSE流 → 解析数据类型 → 分发处理
                                      ↓              ↓
                                   text块         command块
                                      ↓              ↓
                                  显示对话        执行绘图
```

---

### 第一层：SSE 通信原理

SSE（Server-Sent Events）是**单向**的服务器推流协议，基于 HTTP 长连接。

```
客户端                          服务端
  |                               |
  |-- GET /api/ai/cad/draw -----> |
  |                               | LLM开始生成...
  |<-- data: {"type":"text"} ---- |  chunk1
  |<-- data: {"type":"text"} ---- |  chunk2
  |<-- data: {"type":"command"} - |  tool被调用
  |<-- data: {"type":"text"} ---- |  chunk3
  |<-- data: [DONE] ------------- |  结束
```

与 WebSocket 的区别：SSE 是单向的，且**自动重连**，天然适合 LLM 流式输出场景。

```javascript
// 浏览器原生 EventSource
const es = new EventSource(`/api/ai/cad/draw?prompt=画一个圆`);

es.onmessage = (e) => {
    console.log(e.data); // 每次收到一个 chunk
};
```

---

### 第二层：Spring AI 的流式输出结构

Spring AI 用 `Flux<ChatResponse>` 返回数据，每个 chunk 包含不同内容：

```
LLM 生成过程中，会交替产生两种 chunk：

chunk1: { type: "text",    content: "好的，我来帮你" }
chunk2: { type: "text",    content: "绘制一个圆形" }
chunk3: { type: "command", command: { name: "CIRCLE", args: ["50,50", "20"] } }
chunk4: { type: "text",    content: "已完成绘制" }
```

但**默认情况下**，Spring AI 的 `.stream().content()` 只返回文本部分，tool call 结果不会自动推到前端。这就是为什么需要自定义处理。

---

### 第三层：后端如何区分 text 和 command

需要在 Controller 层拦截 `ChatResponse`，判断是文本 chunk 还是 tool call：

```java
@GetMapping(value = "/draw", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<ServerSentEvent<String>> draw(@RequestParam String prompt) {
    
    return chatClient.prompt()
        .user(prompt)
        .tools(cadTools)
        .stream()
        .chatResponse()  // 拿完整 ChatResponse，而不是只拿 content
        .flatMap(response -> {
            List<ServerSentEvent<String>> events = new ArrayList<>();
            
            AssistantMessage msg = response.getResult().getOutput();
            
            // 情况1：普通文本 chunk
            if (msg.getText() != null && !msg.getText().isEmpty()) {
                Map<String, Object> textEvent = Map.of(
                    "type", "text",
                    "content", msg.getText()
                );
                events.add(ServerSentEvent.builder(toJson(textEvent)).build());
            }
            
            // 情况2：Tool Call（LLM 决定调用绘图工具）
            if (msg.getToolCalls() != null) {
                for (AssistantMessage.ToolCall call : msg.getToolCalls()) {
                    // 解析工具参数，构造 CadCommand
                    CadCommand cmd = buildCommand(call);
                    Map<String, Object> cmdEvent = Map.of(
                        "type", "command",
                        "command", cmd
                    );
                    events.add(ServerSentEvent.builder(toJson(cmdEvent)).build());
                }
            }
            
            return Flux.fromIterable(events);
        });
}
```

### 第四层：前端完整处理逻辑

```javascript
// CadAiService.js
export function sendCadPrompt(prompt, onCommand, onText, onDone) {
    
    // 1. 建立 SSE 连接
    const url = `/api/ai/cad/draw?prompt=${encodeURIComponent(prompt)}`;
    const es = new EventSource(url);
    
    // 2. 处理每个 chunk
    es.onmessage = (e) => {
        if (e.data === '[DONE]') {
            es.close();
            onDone?.();
            return;
        }
        
        const data = JSON.parse(e.data);
        
        // 3. 按类型分发
        switch (data.type) {
            case 'text':
                onText(data.content);      // 更新对话框文字
                break;
            case 'command':
                onCommand(data.command);   // 触发绘图
                break;
        }
    };
    
    es.onerror = (err) => {
        console.error('SSE error', err);
        es.close();
    };
    
    // 返回关闭函数，供组件卸载时清理
    return () => es.close();
}
```

```jsx
// AiAssistant.jsx
function AiAssistant({ viewer }) {
    const [messages, setMessages] = useState([]);
    const [input, setInput] = useState('');
    const [aiReply, setAiReply] = useState('');  // 流式积累文本
    
    const handleSend = () => {
        // 追加用户消息
        setMessages(prev => [...prev, { role: 'user', content: input }]);
        setAiReply('');
        
        const cleanup = sendCadPrompt(
            input,
            
            // onCommand: 收到绘图命令
            async (command) => {
                // command = { name: 'CIRCLE', args: ['50,50', '20'] }
                await viewer.execute(command.name, command.args);
            },
            
            // onText: 收到文字 chunk，流式拼接
            (chunk) => {
                setAiReply(prev => prev + chunk);
            },
            
            // onDone: 全部结束，固化消息
            () => {
                setMessages(prev => [...prev, { role: 'assistant', content: aiReply }]);
            }
        );
        
        setInput('');
        return cleanup;
    };
    
    return (
        <div>
            {messages.map((m, i) => (
                <div key={i} className={m.role}>
                    {m.content}
                </div>
            ))}
            
            {/* 流式输出区域 */}
            {aiReply && <div className="assistant streaming">{aiReply}</div>}
            
            <input value={input} onChange={e => setInput(e.target.value)} />
            <button onClick={handleSend}>发送</button>
        </div>
    );
}
```

### 第五层：命令执行的时序问题

这是最容易踩坑的地方。SSE 是异步的，多条命令可能**同时**到达：

```
// 危险：并发执行可能导致命令状态机混乱
es.onmessage = (e) => {
    viewer.execute(cmd.name, cmd.args); // ❌ 没有 await，命令会互相干扰
};
```

需要一个**命令队列**串行执行：

```javascript
class CommandQueue {
    constructor(viewer) {
        this.viewer = viewer;
        this.queue = [];
        this.running = false;
    }
    
    push(command) {
        this.queue.push(command);
        if (!this.running) this.flush();
    }
    
    async flush() {
        this.running = true;
        while (this.queue.length > 0) {
            const cmd = this.queue.shift();
            await this.viewer.execute(cmd.name, cmd.args); // 串行等待
        }
        this.running = false;
    }
}

// 使用
const queue = new CommandQueue(viewer);

sendCadPrompt(prompt,
    (command) => queue.push(command),  // 入队，自动串行执行
    (chunk) => setAiReply(prev => prev + chunk)
);
```

### 完整时序图

```
用户点击发送
    │
    ├─ 前端建立 EventSource 连接
    │
    │  Spring AI 开始调用 LLM
    │  LLM 流式输出...
    │
    ├─ SSE chunk: {type:"text", content:"好的"}
    │   └─ setAiReply 更新 UI
    │
    ├─ SSE chunk: {type:"command", command:{name:"CIRCLE",...}}
    │   └─ queue.push(command)
    │       └─ viewer.execute("CIRCLE", ["50,50","20"])  ← 异步等待完成
    │
    ├─ SSE chunk: {type:"text", content:"圆形已绘制完成"}
    │   └─ setAiReply 继续追加
    │
    └─ SSE: [DONE]
        └─ es.close(), 固化消息记录
```

核心就三点：**SSE 接收流**、**按 type 分发**、**命令队列串行执行**。
