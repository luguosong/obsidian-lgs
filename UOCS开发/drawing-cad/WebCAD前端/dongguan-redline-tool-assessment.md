## 二、需求功能清单及分析

根据客户截图，东莞红线工具包含六大模块：

---

### 模块 1 — 数据调入管理面板

#### 1.1 调入数据面板（分类红线导入）

**需求描述**：以分类方式显示 GIS 库中各类红线（用地红线、道路红线、河道蓝线等），按规划范围批量框选导入到 CAD 图形指定位置。

**现状**：现有 `ImportGisModal` 已支持按图层勾选导入，但缺乏：
- 红线专用分类树（按业务类型归组，而非按 GIS 图层）
- 框选范围过滤（Spatial Filter：只导入宗地范围内的要素）
- "指定插入位置"（CAD 侧坐标偏移对齐）

**难度**：★★★☆☆ 中等  
**依赖**：ArcGIS FeatureServer 空间查询（`geometry` 参数）、复用 `ImportGisModal`

---

#### 1.2 调入地图影像图（分幅/分类显示）

**需求描述**：支持按指定范围从分幅地图影像（标准地形图 1:500~1:2000）导入，支持卫星/规划/影像等不同分类底图切换。

**现状**：现有 `fetchAndInsertMapImage` 支持全幅和视口范围拉取，但缺乏：
- 标准分幅格网（1:500/1:1000/1:2000 国标图幅划分）
- 多图层分类切换（卫星图 / 规划图 / 地形图 来源不同服务）
- 分幅影像本地缓存

**难度**：★★★☆☆ 中等  
**依赖**：`GisServiceTab` 底图模块扩展、国标图幅划分算法

---

#### 1.3 红线检测

**需求描述**：自动检测指定图层上的土地红线，也可手工点击/框选生成红线包络范围。

**现状**：完全未实现。需要：
- WASM 侧新增 `REDLINE_DETECT` 命令（基于图层名过滤 + 闭合多段线/面识别）
- JS 侧框选交互（已有部分选择模式基础）
- 红线实体自动标记（写入自定义扩展数据 XData）

**难度**：★★★★☆ 较高  
**依赖**：ODA WASM 命令开发、XData 扩展属性机制

---

#### 1.4 红线管理

| 子功能 | 说明 | 难度 |
|--------|------|------|
| 红线删除 | 从 DWG 删除实体 + 回写 GIS 删除标记（复用 `GIS_PUSH` 机制） | ★★☆☆☆ 低 |
| 引线坐标标注 | 给红线顶点添加带坐标值的引线注记（自定义 WASM 命令） | ★★★☆☆ 中等 |
| 标注比例 | 控制注记文字与引线的比例因子（标注样式 DIMSCALE） | ★★☆☆☆ 低 |

### 模块 2 — 辅助注释工具

对 GIS 红线实体及其他已加载要素进行 DWG 内标注。

| 子功能 | 技术路径 | 难度 |
|--------|----------|------|
| 引线坐标标注 | 自定义 WASM 命令：选点 → 读坐标 → 插入 LEADER + MTEXT | ★★★☆☆ 中等 |
| 删除坐标标注 | 按图层或选择集批量删除注记实体 | ★★☆☆☆ 低 |
| 多边标注 | 标注多边形各顶点坐标，自动排列文字 | ★★★☆☆ 中等 |
| 面积标注 | 读取 `getArea()` → 格式化 → 插入 MTEXT 到形心 | ★★☆☆☆ 低 |
| 标注长度 | 读取 `getDistAtParam(getEndParam())` → 插入到线中点 | ★★☆☆☆ 低 |
| 属性标注 | 读取 GIS XData 属性字段 → 生成引线文字 | ★★★☆☆ 中等 |
| 属性引线标注 | 多属性字段汇总成表格式引线注记 | ★★★★☆ 较高 |

> **技术注意**：ODA SDK 中曲线长度必须用 `curve.getDistAtParam(curve.getEndParam())` 计算，**不存在 `.length()` 方法**。

### 模块 3 — 红线出图

| 子功能 | 说明 | 难度 |
|--------|------|------|
| 红线置顶 | 调整红线实体的绘制顺序（DrawOrder）置于最上层 | ★★☆☆☆ 低 |
| 文字置顶 | 将注记文字图层 DrawOrder 置顶 | ★★☆☆☆ 低 |
| 随图画框 | **最复杂**：自动生成标准图框（标题栏、图例、比例尺、指北针、坐标网格），并按出图范围裁剪影像 | ★★★★★ 高 |

> **随图画框** 的难点：
> - 国标图框模板（A3/A2 不同图幅规格）
> - 坐标格网注记（每隔 N 米打格网）
> - 图例自动生成（从图层样式映射生成图例框）
> - 比例尺绘制（图形比例尺 + 数字比例尺）
> - 指北针（标准符号插入）
> - 与现有打印/输出流程集成

### 模块 5 — 后台管理

**需求描述**：管理入库图层定义及与图层关联的属性字段配置。

**分析**：这是一个独立的 **管理后台 Web 界面**，需要：
- 后端数据库（PostgreSQL 或 SQLite）存储图层/属性配置
- RESTful 管理 API（增删改查图层定义）
- 前端管理界面（表格 + 表单）
- 与 GIS 服务的图层元数据同步

**难度**：★★★☆☆ 中等（技术上不复杂，但涉及数据库设计）

## 五、技术风险点

| 风险 | 描述 | 缓解措施 |
|------|------|---------|
| **GIS 服务不稳定** | 东莞规划局 ArcGIS 服务 `10.147.2.74:6080` 内网访问，生产部署需 VPN 或反向代理 | 尽早联调；设计离线缓存机制 |
| **坐标系精度** | 东莞使用 EPSG:4526（CGCS2000 第38带），GIS ↔ CAD 转换精度需亚厘米级 | 使用 GDAL WASM 侧 OGR 转换，测量控制点验证 |
| **随图画框复杂度** | 不同比例尺下图框规格、格网间距、字体大小差异大，需与客户反复对样 | 先做原型确认样式，再参数化 |
| **WASM 命令开发** | 红线检测、坐标标注需修改 C++ DrawingWeb 侧 WASM 绑定，编译周期长 | 优先用 JS 侧 AI Code 实现，WASM 侧只做必要扩展 |
| **属性数据同步** | 红线属性（宗地号、地类、面积）存在 GIS，需持久化到 DWG XData 且双向同步 | 采用 GIS ObjectID 作为唯一键绑定 XData |
