# 更新日志

本页整理了 [[Claude Code]] 各版本的重要更新。数据来源于 [GitHub Releases](https://github.com/anthropics/claude-code/releases)。

💡 建议保持 [[Claude Code]] 更新到最新版本，新版本通常包含性能改进和 Bug 修复：

``` bash
npm update -g @anthropic-ai/claude-code
```

## 📦 2.1.144（2026-05-19）

### ✨ 新功能

- **后台会话 `/resume` 支持** — 通过 `claude --bg` 或 agent view 启动的会话现在与交互式会话一同显示，并标记 `bg`
- **后台 subagent 完成通知显示耗时**（例如 "Agent completed · 3h 2m 5s"）
- **`/plugin` 浏览和发现面板显示插件最后更新时间**
- **`/model` 仅切换当前会话的模型** — 在模型选择器中按 `d` 可为新会话设置默认模型

### 🔧 改进

- 将 "extra usage" 重命名为 "usage credits"；`/extra-usage` 更名为 `/usage-credits`（旧名称仍可用）
- 减少了 VS Code 中微调动画颜色数量，降低终端渲染闪烁
- `claude agents` / `--bg` 拒绝消息现在会说明具体原因（非 TTY、环境变量或设置项），而非通用提示
- `claude --bg --name <label>` 现在在启动确认中回显名称
- `claude agents`：使用 Ctrl+R 重命名后台会话后，立即更新已连接会话的横幅
- 后台会话 worktree 隔离保护现在也适用于配置了 `WorktreeCreate` 钩子的非 git VCS 用户
- Plugin marketplace 添加/更新现在遵循 `CLAUDE_CODE_PLUGIN_PREFER_HTTPS`
- `/plugin` 在启用、禁用或卸载插件后现在返回已安装列表
- `/doctor` 现在在 command hook 缺少 `command` 字段时显示 exec-form 示例
- Skill 列表截断不再作为启动通知显示 — 运行 `/doctor` 查看完整详情
- 改进了罕见的 pre-response 流式停滞恢复 — 现在重试一次流式请求，而不是回退到更慢的非流式请求
- 改进了 SDK/headless MCP 启动：预等待现在与启动过程并行，不再阻塞首轮调用（慢速 MCP 服务器场景下最多快 2 秒）
- 调查问卷后续提示现在在每次非关闭的回复后出现，文案根据上下文调整，方便通过 `/feedback` 分享更多细节
- `/bg` 和 `←` 分离现在保留通过 `/add-dir` 添加的目录
- 恢复的会话现在保留其使用的模型，而不是采用其他会话的 `/model` 选择
- `claude respawn <id>` 对已停止的后台会话现在正确启动运行，而非显示 "stopped"
- 后台 Bash 任务由 subagent 生成时，进程退出后不再在 SDK 任务面板中保持 "Running" 状态
- 已完成或已停止的后台会话唤醒时短暂失败不再被永久标记为启动崩溃
- `claude agents` 附加会话中的 Markdown 链接现在渲染为可点击的超链接，而非纯文本
- 自定义 `spinnerVerbs` 现在仅应用于轮次中消息 — 过去式内置动词如 "Worked for 5s" 已恢复

### 🐛 修复

- 修复了当 `api.anthropic.com` 不可达时（强制门户、防火墙、VPN 问题）启动挂起长达 75 秒的问题 — 侧信道 API 调用现在 15 秒后超时
- 修复了窗口尺寸变更事件丢失后（例如拖动 VS Code 分隔面板）终端输出乱码的问题 — 现在在下一帧自动恢复，无需 Ctrl+L
- 修复了超长会话中可能出现的渐进式终端显示损坏（残留/乱码字符），之前只能通过调整终端大小或重启来清除
- 修复了 macOS 上项目位于全磁盘访问保护文件夹下时，后台会话以 "exit 1 before init" 崩溃的问题（2.1.143 回归）
- 修复了读取文件扩展名与实际内容不匹配的图片文件（例如 HTML 保存为 .png）时导致会话不可恢复的问题 — 现在回退为文本模式
- 减少了搜索时的误报工具错误：`head`/`tail` 文件视图现在满足编辑前读取检查，`egrep`、`fgrep`、`git grep` 或 `git diff` 返回 "无匹配"（退出码 1）不再被报告为命令失败
- 修复了进入 worktree 或某些后台会话后 `/branch` 报错 "No conversation to branch" 的问题
- 修复了在 AskUserQuestion 备注字段按 Escape 会中止当前轮次而非返回答案选择的问题
- 修复了通过 IDE 模型选择器或 `applyFlagSettings` 更改模型后不生效的问题
- 修复了 Bedrock 和 Vertex 用户无法从 `/model` 选择器选择 "Opus (1M context)" 的问题（v2.1.129 回归）
- 修复了设置了 `forceLoginMethod` 和 `forceLoginOrgUUID` 的用户在远程会话登录时失败并提示 "Can't access this organization" 的问题
- 修复了 [[MCP 服务器]] 分页 `tools/list` 响应仅返回第一页，静默丢弃工具的问题
- 修复了不支持 MIME 类型的 MCP 图片（如 SVG）导致会话中断的问题 — 现在保存到磁盘并在工具结果中引用
- 修复了 skill 目录内运行构建时文件描述符耗尽的问题 — 非 `.md` 文件不再触发 skill 重载
- 修复了会话标题从 plugin monitor 输出生成而非用户首次提示的问题
- 修复了 Skill 工具在 headless 模式下报权限错误的问题（v2.1.141 回归）
- 修复了在自己的设置中启用的插件在全新机器首次加载后显示 "not cached" 错误的问题；仅通过项目 `.claude/settings.json` 启用的插件现在显示可操作的 `claude plugin install` 提示
- 修复了 `claude mcp list` 在 `.mcp.json` 无法解析时（例如使用 VS Code 的 `"servers"` 键而非 `"mcpServers"`）静默报告无服务器的问题 — 现在显示配置错误
- 修复了自定义 `ANTHROPIC_BASE_URL` 和 Bedrock Mantle 上后台 side-query 未使用 Haiku 的问题 — 现在在配置了第一方 API 密钥或未设置 Haiku 模型时正确回退
- 修复了 Windows 上附加后台会话中的滚动问题 — PgUp/PgDn、鼠标滚轮和 Ctrl+O 转录导航现在正常工作
- 修复了附加到后台会话时关闭终端导致的崩溃
- 修复了 `! <cmd>` 执行会话附加时不响应 Ctrl+C 的问题 — 现在会中断正在运行的命令
- 修复了 agent view 中 shell 命令行完成后仍停留在 Working 状态，以及输出过期后按 Enter 会重新运行命令的问题
- 修复了 Windows 上在 `claude agents` 中按 `←` 导致列表不再响应键盘输入的问题
- 修复了 Windows Terminal 中 CJK 内容下切换 Agent View 面板时左侧出现幽灵字符的问题
- 修复了分离已在原地编辑的会话后，Edit/Write 立即拒绝并提示 "background session hasn't isolated its changes yet" 的问题
- 修复了 `/resume` 选择器不显示从后台会话 fork 出的会话的问题
- 修复了从 `claude agents` 打开会话或运行 `claude logs <id>` 时，后台服务无响应导致挂起的问题 — 现在 10 秒后超时并显示恢复提示
- 修复了自定义 `spinnerVerbs` 应用到轮次完成持续时间消息的问题 — 过去式内置消息如 "Worked for 5s" 已恢复

## 📦 2.1.143（2026-05-19）

### ✨ 新功能

- **Plugin 依赖强制执行** — `claude plugin disable` 在另一个已启用插件依赖目标插件时拒绝操作（附带可复制的禁用链提示），`claude plugin enable` 自动强制启用传递依赖
- **`/plugin` marketplace 浏览面板显示预估上下文成本**（每轮和每次调用的 token 估算）
- **新增 `worktree.bgIsolation: "none"` 设置** — 允许后台会话直接编辑工作副本而无需 `EnterWorktree`，适用于 worktree 不切实际的仓库
- **PowerShell 工具现在传递 `-ExecutionPolicy Bypass`** — 可通过 `CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1` 取消
- **后台会话从空闲唤醒后保留设定的模型和 effort 级别**
- **附加 agent 会话中 Shift+Tab 现在将 auto 模式纳入循环**
- **PowerShell 工具对 Windows 上的 Bedrock、Vertex 和 Foundry 用户默认启用** — 可通过 `CLAUDE_CODE_USE_POWERSHELL_TOOL=0` 取消

### 🔧 改进

- `claude agents` 现在接受 `--add-dir`、`--settings`、`--mcp-config` 和 `--plugin-dir`，并应用于仪表板和从中分发的后台会话
- `claude agents` 接受 `--permission-mode`、`--model`、`--effort` 和 `--dangerously-skip-permissions` 来设置从视图中分发会话的默认值
- `claude --bg --dangerously-skip-permissions` 现在在 retire→wake 后持久保留
- `/bg` 现在保留 `--mcp-config`、`--settings`、`--add-dir`、`--plugin-dir` 和 `--strict-mcp-config`，因此后台会话在 respawn 后保留其 [[MCP 服务器]] 和设置
- 从 `claude agents` 启动的后台会话现在遵循 settings.json 中的 `permissions.defaultMode`（之前被覆盖为 auto 模式）
- `/bg` 和 `←` 分离现在保留 `--fallback-model`，因此后台 worker 在过载时降级到 fallback 模型而非硬失败
- `/bg` 和 `←` 分离现在保留 `--allow-dangerously-skip-permissions`，因此 fork 的 worker 在其 Shift+Tab 循环中保持绕过权限可用

### 🐛 修复

- 修复了损坏的 `.credentials.json`（`scopes` 值非数组）导致 CLI 启动挂起或静默中止 OAuth token 刷新的问题
- 修复了 Windows Terminal 和 WSL 上 `claude agents` 中右键粘贴不工作的问题
- 修复了 stop hooks 反复阻止导致无限循环的问题 — 连续阻止 8 次后轮次现在以警告结束（可通过 `CLAUDE_CODE_STOP_HOOK_BLOCK_CAP` 覆盖）
- 修复了 Claude 空闲等待下一轮 `/loop` 迭代时，Esc/Ctrl+C 无法取消待处理的 `/loop` 唤醒的问题
- 修复了后台 shell 或委派的 subagent 仍在运行时 `/goal` 评估器触发的问题
- 修复了 settings.json `env` 中的 `NO_COLOR`/`FORCE_COLOR` 剥离 [[Claude Code]] 自身 UI 颜色的问题 — 现在仅应用于子进程
- 修复了 Windows 上 agent view 列出会话时重复生成 PowerShell 进程的问题
- 修复了不带提示的 `/bg` 向 fork 的会话发送 "continue" 的问题 — fork 现在等待输入
- 修复了 `--agent <name>` 不带 `plugin:` 前缀时找不到 plugin 提供的 agent 的问题
- 修复了从 agent view 删除会话后未删除其转录文件的问题
- 修复了 Windows Terminal 中附加后台会话滚动时残留片段渲染的问题
- 修复了主机休眠或 macOS App Nap 后后台 agent 误报 worker-stall 检测风暴的问题
- 修复了 5xx 错误消息指向 status.claude.com 而非指明配置的网关或云提供商的问题
- 修复了后台会话静默捕获 IDE 文件引用到 warm spare 输入中，导致引用被前置到从 `claude agents` 分发的下一个提示的问题
- Worktree 清理在 `git worktree remove` 失败时不再回退到 `rm -rf`，防止丢失 gitignored 或进行中的文件
- 修复了 macOS 上后台任务会话在 `~/Documents`、`~/Desktop` 或 `~/Downloads` 下读取文件时出现 "Operation not permitted" 错误的问题，即使已授予全磁盘访问权限
- 修复了 Windows 上在响应流式传输期间按 `←` 可能导致 agents 列表对所有输入不再响应的问题
- 修复了后台 daemon spawn 在 `~/.local/bin/claude` 启动器缺失或不可执行时未回退到正在运行的二进制文件的问题
- 修复了 `claude agents --allow-dangerously-skip-permissions` 将分发的会话默认设为 bypass 模式而非将其设为权限循环中可用的选项的问题

## 📦 2.1.142（2026-05-15）

### ✨ 新功能

- **`claude agents` 新增命令行标志**：`--add-dir`、`--settings`、`--mcp-config`、`--plugin-dir`、`--permission-mode`、`--model`、`--effort` 和 `--dangerously-skip-permissions`，用于配置分发的后台会话
- **Fast 模式默认使用 Opus 4.7**（此前为 Opus 4.6）。设置 `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1` 可固定使用 Opus 4.6

### 🔧 改进

- 包含根级 `SKILL.md` 且无 `skills/` 子目录的 [[plugin|插件]] 现在会被识别为 skill
- `/plugin` 详情面板和 `claude plugin details` 现在显示插件提供的 LSP 服务器
- `/web-setup` 在替换已有的 GitHub App 连接前会发出警告
- **改进响应式压缩**：首次摘要尝试现在从原始请求的溢出大小开始，避免浪费一次接近满上下文的重试
- **改进 Hook 配置错误提示**：为 `SessionStart`/`Setup`/`SubagentStart` 配置 prompt 或 agent 类型 Hook 时，现在会显示明确的 "请改用 command 类型 Hook" 错误提示
- 移除了 Usage Policy 拒绝消息中过时的 `/model claude-sonnet-4-20250514` 建议

### 🐛 修复

- `MCP_TOOL_TIMEOUT` 未提高远程 HTTP 和 SSE [[MCP 服务器]] 的单次请求超时时间，导致工具调用始终被限制在 60 秒
- 后台会话无法识别已有的 git [[worktree]]，导致 Edit 被阻止而 [[EnterWorktree]] 拒绝创建重复项
- macOS 休眠/唤醒后后台会话消失且守护进程重连失败——守护进程现在通过检测时钟跳变而非将其视为经过的空闲时间
- 二进制文件升级后（如 `brew upgrade`）守护进程未干净退出，导致分发的 agent 在已删除路径上崩溃循环
- [[Claude-in-Chrome]] 扩展在无共享标签页时连接，导致后台 agent 崩溃循环
- 在已连接的 `claude agents` 会话中点击链接——后台 worker 的无头浏览器 shim 不再在连接状态下生效
- `claude agents` 中 "v 在编辑器中打开" 使用了守护进程的默认编辑器而非 shell 的 `$EDITOR`/`$VISUAL`
- `claude agents` 在 Windows 网络驱动器工作目录上死锁；Ctrl+C 现在可在启动期间使用
- 从 Apple Terminal 或其他仅 256 色终端连接 `claude agents` 会话时的背景色溢出问题
- `claude --bg --dangerously-skip-permissions` 未在 retire/wake 周期中持久化
- 首条消息为链接时，会话标题从 URL 派生
- 远程客户端发送的冗余 `set_model` 请求在对话记录中注入重复的 `/model` 面包屑
- [[plugin|插件]] 使用 `skills: ["./"]` 时显示虚假的 "路径逃逸插件目录" 错误
- 插件缓存清理在缺少安装元数据时删除了活动插件版本目录
- `/plugin` 浏览面板对新发布的插件显示 "0 次安装"
- 插件安全通告未列出所有覆盖默认文件夹的 `plugin.json` 键

## 📦 2.1.141（2026-05-14）

### ✨ 新功能

- **Hook `terminalSequence` 字段**：Hook JSON 输出新增 `terminalSequence` 字段，Hook 可以在没有控制终端的情况下发送桌面通知、窗口标题和响铃
- **`CLAUDE_CODE_PLUGIN_PREFER_HTTPS` 环境变量**：通过 HTTPS 而非 SSH 克隆 GitHub 插件源，适用于没有 GitHub SSH 密钥的环境
- **`ANTHROPIC_WORKSPACE_ID` 环境变量**：用于工作负载身份联合——当联合规则覆盖多个工作区时，将生成的令牌限定到特定工作区
- **`claude agents --cwd <path>`**：将会话列表限定到指定目录
- **`/feedback` 支持包含近期会话**：可以包含最近 24 小时或 7 天的会话，适用于跨会话的问题反馈
- **Rewind 菜单新增 "Summarize up to here"**：压缩较早的上下文，同时保留最近的对话轮次
- **Auto 模式权限对话框解释 `permissions.ask` 规则**：当 `permissions.ask` 规则触发提示时，对话框会解释原因
- **恢复 "在 IDE 中查看 diff" 选项**：当 IDE 已连接时，文件编辑权限提示中重新显示该选项

### 🔧 改进

- 通过 `/bg` 或 `←←` 启动的后台 [[Agent]] 现在保留当前权限模式而非恢复默认
- `claude agents`：完成工作但留有后台 shell 运行的 agent 现在移至 Completed 而非停留在 Working
- **改进长时间思考期间的 spinner 反馈**：spinner 在 10 秒后变为琥珀色，表示 [[Claude Code]] 仍在工作中
- **改进插件菜单导航**：`→`/Tab 切换标签页，`↑` 移动到标签栏，全屏模式下标签标题和搜索框可点击

### 🐛 修复

- 后台 side-queries 在 Bedrock/Vertex/Foundry/gateway 上发送不可用的 Haiku 模型 ID（未设置 `ANTHROPIC_SMALL_FAST_MODEL` 覆盖时）——现在回退到主循环模型
- `claude daemon status` 和 `/doctor` 在 Windows 上当守护进程管道密钥文件被锁定或不可读时抛出异常——现在显示底层错误而非不透明的失败信息
- `claude agents` 通过添加标志的包装器启动时显示 agent 类型列表而非仪表板
- `claude agents` 打开崩溃的会话在工作目录被删除时触发冗余分发
- 自定义 `ANTHROPIC_BASE_URL` 网关上的后台任务未自动命名——命名器现在在未配置 Haiku 模型时使用主模型
- `/model` 在一个会话中静默更改其他并发会话的自动压缩阈值
- 工具权限提示打开时切换权限模式未在新设置允许工具时自动关闭提示
- 权限/对话框提示打开时按 Enter 同时提交了输入框中的文本
- Hook 在 [[EnterWorktree]] 切换工作目录后收到不存在的 `transcript_path`
- 包含单元格换行的 Markdown 表格回退到垂直键值布局而非渲染为带边框的网格（v2.1.136 引入的回归）
- 已取消的提示在自动恢复到输入框时从 Up-arrow 历史中移除，导致重复条目
- 在任何响应前用 Ctrl+C/Esc 取消的提示从 Up-arrow 历史中丢失
- Ctrl+C 无法在 vim INSERT/VISUAL 模式下中断正在运行的对话轮次
- 替代 `chat:submit` 键绑定（如 `meta+enter`、`ctrl+enter`）在 `enter` 被重新绑定为 `chat:newline` 时不工作
- 配置了输出样式时提示建议被静默禁用
- `spinnerVerbs` 设置在对话完成消息中未被使用
- AskUserQuestion 弹窗隐藏了前一条聊天内容的最后一行
- Web Search 状态在搜索返回错误时显示 "Did 0 searches"
