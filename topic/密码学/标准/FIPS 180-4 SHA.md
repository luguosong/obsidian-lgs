# FIPS 180-4 安全哈希标准（SHS）

> [!info] 标准信息
> - **全称**：Federal Information Processing Standards Publication 180-4
> - **类别**：计算机安全标准 - 密码学
> - **发布机构**：NIST（美国国家标准与技术研究院）
> - **发布日期**：2015 年 8 月
> - **替代**：FIPS 180-3（2008 年 10 月 31 日）
> - **DOI**：[10.6028/NIST.FIPS.180-4](http://dx.doi.org/10.6028/NIST.FIPS.180-4)

## 1. 概述

本标准规定了以下安全哈希算法：SHA-1、SHA-224、SHA-256、SHA-384、SHA-512、SHA-512/224 和 SHA-512/256。

所有算法都是**迭代型单向哈希函数**，可以将消息处理为称为"消息摘要"的压缩表示。这些算法能够验证消息的完整性：对消息的任何修改，都会以极高的概率产生不同的消息摘要。

每个算法分为两个阶段：
1. **预处理**：填充消息、解析为 m-bit 块、设置初始值
2. **哈希计算**：从填充后的消息生成消息调度，使用函数、常量和字运算迭代生成哈希值序列

### 1.1 算法基本属性

| 算法 | 消息大小（bits） | 块大小（bits） | 字大小（bits） | 摘要大小（bits） |
|------|:-:|:-:|:-:|:-:|
| SHA-1 | < 2^64 | 512 | 32 | 160 |
| SHA-224 | < 2^64 | 512 | 32 | 224 |
| SHA-256 | < 2^64 | 512 | 32 | 256 |
| SHA-384 | < 2^128 | 1024 | 64 | 384 |
| SHA-512 | < 2^128 | 1024 | 64 | 512 |
| SHA-512/224 | < 2^128 | 1024 | 64 | 224 |
| SHA-512/256 | < 2^128 | 1024 | 64 | 256 |

### 1.2 安全性说明

这些哈希算法被称为"安全"的，因为对于给定算法：

1. **找到与给定摘要对应的消息**在计算上不可行（抗原像攻击）
2. **找到两个产生相同摘要的不同消息**在计算上不可行（抗碰撞攻击）

对消息的任何修改，都会以极高概率产生不同的消息摘要。

| 符号 | 运算 |
|------|------|
| ∧ | 按位与（AND） |

| ⊕ | 按位异或（XOR） |

| « | 左移位 |
| » | 右移位 |

**循环移位运算**：

- **ROTL^n(x)** = (x « n) ∨ (x » (w-n)) — 循环左移
- **ROTR^n(x)** = (x » n) ∨ (x « (w-n)) — 循环右移
- **SHR^n(x)** = x » n — 右移位

等价关系：ROTL^n(x) ≈ ROTR^(w-n)(x)

## 3. 记法与约定

### 3.1 位串与整数

- 十六进制数字集合：{0, 1, …, 9, a, …, f}，每个十六进制数字表示 4-bit 串
- 使用**大端序**（big-endian）：每个字中最高有效位存储在最左位置
- 整数表示：
  - 0 到 2^32-1 的整数可用 32-bit 字表示
  - 0 到 2^64-1 的整数可用 64-bit 字表示
  - 0 到 2^64 的整数 Z = 2^32·X + Y，表示为字对 (x, y)（用于 SHA-1/224/256）
  - 0 到 2^128 的整数 Z = 2^64·X + Y，表示为字对 (x, y)（用于 SHA-384/512 等）

### 3.2 消息块大小

- **SHA-1、SHA-224、SHA-256**：512 bits = 十六个 32-bit 字
- **SHA-384、SHA-512、SHA-512/224、SHA-512/256**：1024 bits = 十六个 64-bit 字

## 4. 函数与常量

### 4.1 函数

#### 4.1.1 SHA-1 函数

SHA-1 使用 80 个逻辑函数 f_0, f_1, …, f_79，每个函数操作三个 32-bit 字 (x, y, z)：

| 轮次范围 t | 函数 f_t(x, y, z) |
|:--|:--|
| 0 ≤ t ≤ 19 | Ch(x, y, z) = (x ∧ y) ⊕ (¬x ∧ z) |
| 20 ≤ t ≤ 39 | Parity(x, y, z) = x ⊕ y ⊕ z |
| 40 ≤ t ≤ 59 | Maj(x, y, z) = (x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z) |
| 60 ≤ t ≤ 79 | Parity(x, y, z) = x ⊕ y ⊕ z |

#### 4.1.2 SHA-224 和 SHA-256 函数

操作 32-bit 字：

| 函数 | 定义 |
|:--|:--|
| Ch(x, y, z) | (x ∧ y) ⊕ (¬x ∧ z) |
| Maj(x, y, z) | (x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z) |
| Σ_0^{256}(x) | ROTR^2(x) ⊕ ROTR^13(x) ⊕ ROTR^22(x) |
| Σ_1^{256}(x) | ROTR^6(x) ⊕ ROTR^11(x) ⊕ ROTR^25(x) |
| σ_0^{256}(x) | ROTR^7(x) ⊕ ROTR^18(x) ⊕ SHR^3(x) |
| σ_1^{256}(x) | ROTR^17(x) ⊕ ROTR^19(x) ⊕ SHR^10(x) |

#### 4.1.3 SHA-384、SHA-512、SHA-512/224、SHA-512/256 函数

操作 64-bit 字：

| 函数 | 定义 |
|:--|:--|
| Ch(x, y, z) | (x ∧ y) ⊕ (¬x ∧ z) |
| Maj(x, y, z) | (x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z) |
| Σ_0^{512}(x) | ROTR^28(x) ⊕ ROTR^34(x) ⊕ ROTR^39(x) |
| Σ_1^{512}(x) | ROTR^14(x) ⊕ ROTR^18(x) ⊕ ROTR^41(x) |
| σ_0^{512}(x) | ROTR^1(x) ⊕ ROTR^8(x) ⊕ SHR^7(x) |
| σ_1^{512}(x) | ROTR^19(x) ⊕ ROTR^61(x) ⊕ SHR^6(x) |

> [!tip] Ch 和 Maj
> `Ch(x,y,z)` 表示"选择"：x 的位为 1 时选择 y 的对应位，否则选择 z 的。`Maj(x,y,z)` 表示"多数"：取三个输入中多数为 1 的位。这两个函数在所有 SHA 算法中都有，只是字大小不同。

## 5. 预处理

预处理包含三个步骤：
1. 填充消息
2. 解析消息为消息块
3. 设置初始哈希值 H^(0)

### 5.1 消息填充

#### 5.1.1 SHA-1、SHA-224 和 SHA-256

设消息长度为 λ bits。依次追加：
1. 一个 "1" bit
2. k 个零 bit，其中 k 是满足 **λ + 1 + k ≡ 448 (mod 512)** 的最小非负整数
3. 64-bit 的消息长度 λ 的二进制表示

填充后消息长度为 512 的倍数。

> [!example] 填充示例
> 消息 "abc"（8-bit ASCII）长度 λ = 24 bits：
> - 追加 1 bit "1"，然后 423 个零 bit（因为 24 + 1 + 423 = 448），最后是 64-bit 长度表示
> - 填充结果：`01100001 01100010 01100011 1 00…00 00…011000`
> - 即 "a" "b" "c" + 填充 + 长度(24)

#### 5.1.2 SHA-384、SHA-512、SHA-512/224 和 SHA-512/256

设消息长度为 λ bits。依次追加：
1. 一个 "1" bit
2. k 个零 bit，其中 k 是满足 **λ + 1 + k ≡ 896 (mod 1024)** 的最小非负整数
3. 128-bit 的消息长度 λ 的二进制表示

填充后消息长度为 1024 的倍数。

### 5.2 消息解析

将填充后的消息解析为 N 个 m-bit 块：

- **SHA-1/224/256**：解析为 N 个 512-bit 块，每块表示为 16 个 32-bit 字 M^(i)_0, …, M^(i)_15
- **SHA-384/512 等**：解析为 N 个 1024-bit 块，每块表示为 16 个 64-bit 字 M^(i)_0, …, M^(i)_15

## 6. 安全哈希算法

> [!note] 算法关系
> - **SHA-224** 与 SHA-256 核心逻辑相同，仅 IV 不同且截断为 224 bits
> - **SHA-384**、SHA-512/224、SHA-512/256 与 SHA-512 核心逻辑相同，仅 IV 不同且截断位数不同

### 6.1 SHA-1

**参数**：消息长度 0 < λ ≤ 2^64 bits

**使用**：
- 80 个 32-bit 字的消息调度（W_0, …, W_79）
- 5 个 32-bit 工作变量（a, b, c, d, e）
- 5 个 32-bit 字的哈希值
- 输出 160-bit 消息摘要

#### 6.1.1 SHA-1 预处理

1. 设置初始哈希值 H^(0)（见 5.3.1）
2. 填充并解析消息（见第 5 节）

#### 6.1.2 SHA-1 哈希计算

加法模 2^32。对每个消息块 M^(1), …, M^(N)：

```
For i = 1 to N:
{
    1. 准备消息调度 {W_t}:
       W_t = M_t^(i)                                    (0 ≤ t ≤ 15)
       W_t = ROTL^1(W_{t-3} ⊕ W_{t-8} ⊕ W_{t-14} ⊕ W_{t-16})  (16 ≤ t ≤ 79)

    2. 初始化工作变量:
       a = H_0^(i-1), b = H_1^(i-1), c = H_2^(i-1)
       d = H_3^(i-1), e = H_4^(i-1)

    3. For t = 0 to 79:
       {
           T = ROTL^5(a) + f_t(b, c, d) + e + K_t + W_t
           e = d
           d = c
           c = ROTL^30(b)
           b = a
           a = T
       }

    4. 计算中间哈希值 H^(i):
       H_0^(i) = a + H_0^(i-1)
       H_1^(i) = b + H_1^(i-1)
       H_2^(i) = c + H_2^(i-1)
       H_3^(i) = d + H_3^(i-1)
       H_4^(i) = e + H_4^(i-1)
}
```

最终 160-bit 消息摘要 = H_0^(N) ‖ H_1^(N) ‖ H_2^(N) ‖ H_3^(N) ‖ H_4^(N)

#### 6.1.3 SHA-1 替代计算方法（节省内存）

使用 16 个字的循环队列代替 80 个字的数组。设 MASK = `0000000f`。

```
For i = 1 to N:
{
    1. For t = 0 to 15: { W_t = M_t^(i) }

    2. 初始化工作变量（同上）

    3. For t = 0 to 79:
       {
           s = t ∧ MASK
           If t ≥ 16:
               W_s = ROTL^1(W_{(s+13)∧MASK} ⊕ W_{(s+8)∧MASK}
                             ⊕ W_{(s+2)∧MASK} ⊕ W_s)
           T = ROTL^5(a) + f_t(b, c, d) + e + K_t + W_s
           e = d, d = c, c = ROTL^30(b), b = a, a = T
       }

    4. 计算中间哈希值（同上）
}
```

> [!tip] 空间换时间
> 替代方法节省 64 个 32-bit 字的存储空间（256 字节），但地址计算更复杂，执行时间可能更长。这是嵌入式系统中常见的权衡。

### 6.2 SHA-256

**参数**：消息长度 0 < λ ≤ 2^64 bits

**使用**：
- 64 个 32-bit 字的消息调度（W_0, …, W_63）
- 8 个 32-bit 工作变量（a, b, c, d, e, f, g, h）
- 8 个 32-bit 字的哈希值
- 2 个临时字 T_1、T_2
- 输出 256-bit 消息摘要

#### 6.2.1 SHA-256 预处理

1. 设置初始哈希值 H^(0)（见 5.3.3）
2. 填充并解析消息

#### 6.2.2 SHA-256 哈希计算

加法模 2^32。对每个消息块 M^(1), …, M^(N)：

```
For i = 1 to N:
{
    1. 准备消息调度 {W_t}:
       W_t = M_t^(i)                                            (0 ≤ t ≤ 15)
       W_t = σ_1^{256}(W_{t-2}) + W_{t-7}
             + σ_0^{256}(W_{t-15}) + W_{t-16}                   (16 ≤ t ≤ 63)

    2. 初始化工作变量:
       a = H_0^(i-1), b = H_1^(i-1), c = H_2^(i-1), d = H_3^(i-1)
       e = H_4^(i-1), f = H_5^(i-1), g = H_6^(i-1), h = H_7^(i-1)

    3. For t = 0 to 63:
       {
           T_1 = h + Σ_1^{256}(e) + Ch(e, f, g) + K_t + W_t
           T_2 = Σ_0^{256}(a) + Maj(a, b, c)
           h = g
           g = f
           f = e
           e = d + T_1
           d = c
           c = b
           b = a
           a = T_1 + T_2
       }

    4. 计算中间哈希值 H^(i):
       H_0^(i) = a + H_0^(i-1), H_1^(i) = b + H_1^(i-1)
       H_2^(i) = c + H_2^(i-1), H_3^(i) = d + H_3^(i-1)
       H_4^(i) = e + H_4^(i-1), H_5^(i) = f + H_5^(i-1)
       H_6^(i) = g + H_6^(i-1), H_7^(i) = h + H_7^(i-1)
}
```

最终 256-bit 消息摘要 = H_0^(N) ‖ H_1^(N) ‖ … ‖ H_7^(N)

### 6.3 SHA-224

SHA-224 与 SHA-256 **完全相同**，仅有两处不同：
1. 初始哈希值 H^(0) 使用 5.3.2 节的值
2. 最终 224-bit 摘要 = H_0^(N) ‖ H_1^(N) ‖ … ‖ H_6^(N)（截断为最左 224 bits）

### 6.4 SHA-512

**参数**：消息长度 0 < λ ≤ 2^128 bits

**使用**：
- 80 个 64-bit 字的消息调度（W_0, …, W_79）
- 8 个 64-bit 工作变量（a, b, c, d, e, f, g, h）
- 8 个 64-bit 字的哈希值
- 2 个临时字 T_1、T_2
- 输出 512-bit 消息摘要

#### 6.4.1 SHA-512 预处理

1. 设置初始哈希值 H^(0)（见 5.3.5）
2. 填充并解析消息

#### 6.4.2 SHA-512 哈希计算

加法模 2^64。对每个消息块 M^(1), …, M^(N)：

```
For i = 1 to N:
{
    1. 准备消息调度 {W_t}:
       W_t = M_t^(i)                                            (0 ≤ t ≤ 15)
       W_t = σ_1^{512}(W_{t-2}) + W_{t-7}
             + σ_0^{512}(W_{t-15}) + W_{t-16}                   (16 ≤ t ≤ 79)

    2. 初始化工作变量:
       a = H_0^(i-1), b = H_1^(i-1), c = H_2^(i-1), d = H_3^(i-1)
       e = H_4^(i-1), f = H_5^(i-1), g = H_6^(i-1), h = H_7^(i-1)

    3. For t = 0 to 79:
       {
           T_1 = h + Σ_1^{512}(e) + Ch(e, f, g) + K_t + W_t
           T_2 = Σ_0^{512}(a) + Maj(a, b, c)
           h = g, g = f, f = e, e = d + T_1
           d = c, c = b, b = a, a = T_1 + T_2
       }

    4. 计算中间哈希值 H^(i):
       H_0^(i) = a + H_0^(i-1), ..., H_7^(i) = h + H_7^(i-1)
}
```

最终 512-bit 消息摘要 = H_0^(N) ‖ H_1^(N) ‖ … ‖ H_7^(N)

### 6.5 SHA-384

SHA-384 与 SHA-512 **完全相同**，仅有两处不同：
1. 初始哈希值 H^(0) 使用 5.3.4 节的值
2. 最终 384-bit 摘要 = H_0^(N) ‖ H_1^(N) ‖ … ‖ H_5^(N)（截断为最左 384 bits）

### 6.6 SHA-512/224

SHA-512/224 与 SHA-512 **完全相同**，仅有两处不同：
1. 初始哈希值 H^(0) 使用 5.3.6.1 节的值
2. 最终 224-bit 摘要截断为最左 224 bits

### 6.7 SHA-512/256

SHA-512/256 与 SHA-512 **完全相同**，仅有两处不同：
1. 初始哈希值 H^(0) 使用 5.3.6.2 节的值
2. 最终 256-bit 摘要截断为最左 256 bits

## 7. 消息摘要截断

如果应用需要不同长度的摘要，可以对较大摘要进行截断，取最左边的适当位数。截断的安全影响参见 [SP 800-107]。
